8.3 Ascend 特定适配点

vllm-ascend 核心适配

  1. CaMemAllocator（Camera Memory Allocator）
    • Ascend NPU 专用内存分配器
    • 支持睡眠模式下的权重卸载和恢复
    • 标签化内存池管理（"weights", "kv_cache"）
  2. 2M 内存对齐
    • 支持 Prefill Disaggregation 特性
    • K/V Cache 地址对齐到 2MB 边界
    • 通过 _align_memory 方法实现
  3. DeepSeek MLA 支持
    • NoPE Cache（kv_lora_rank）和 RoPE Cache（qk_rope_head_dim）分离
    • 显著减少 KV Cache 内存占用
    • 适配 DeepSeek V2/V3/V3.2 等模型
  4. 稀疏注意力支持
    • DeepSeek V3.2 特有的 DSA K Cache
    • 三元组 (K, V, DSA_K) 存储结构
    • 根据稀疏比例分配内存

8.4 关键设计决策

为什么使用一维 int8 缓冲区？

  • 精确控制：按字节精确控制分配的内存大小
  • 灵活重塑：可以重塑为任意形状和 dtype
  • 对齐支持：便于实现 2M 内存对齐

为什么需要 AttentionGroup？

  • 类型分组：将相同类型的 Attention Backend 分组管理
  • 统一优化：对同组 Backend 应用统一的优化策略
  • 混合块支持：支持不同层使用不同的 block size

为什么 K/V Cache 要分离？

  • Prefill Disaggregation：支持 Prefill 和解码阶段分离
  • MLA 支持：DeepSeek 模型需要 NoPE/RoPE 分离
  • 稀疏注意力：DeepSeek V3.2 需要 DSA K Cache

8.6 结语

通过本文档的分析，我们深入了解了 vLLM Ascend 中 KV Cache 初始化的完整流程。从 NPUWorker.initialize_from_config 入口开始，经过 Attention Backend 初始化、KV Cache 张量分配、重塑、绑定和注册，最终完成了整个初始化过程。

vllm-ascend 项目针对 Ascend NPU 进行了大量适配工作，包括专用内存分配器、2M 内存对齐、DeepSeek MLA 和稀疏注意力支持等。这些适配使得 vLLM Ascend 能够高效地在华为 Ascend NPU 上运行各种大语言模型。
