06 - KV Cache 绑定与注册

在 KV Cache 张量分配和重塑完成后，需要将这些张量绑定到模型的前后文中，以便在推理过程中使用。此外，如果启用了 KV Transfer 功能，还需要注册 KV Cache 到传输组。

6.1 initialize_kv_cache_tensors 完成流程

vllm_ascend/worker/model_runner_v1.py:2471
def initialize_kv_cache_tensors(self, kv_cache_config: KVCacheConfig) -> dict[str, torch.Tensor]:
    """
    Initialize the memory buffer for KV cache.

    Args:
        kv_cache_config: The KV cache config
    Returns:
        Dict[str, torch.Tensor]: A map between layer names to their
        corresponding memory buffer for KV cache.
    """
    # 1. 初始化 KV Cache 内存缓冲区
    kv_cache_raw_tensors = self._allocate_kv_cache_tensors(kv_cache_config)

    # 2. 将缓冲区重塑为所需的形状
    kv_caches = self._reshape_kv_cache_tensors(kv_cache_config, kv_cache_raw_tensors)

    # 3. 设置跨层 KV Cache 共享
    for layer_name, target_layer_name in self.shared_kv_cache_layers.items():
        logger.debug("%s reuses KV cache of %s", layer_name, target_layer_name)
        kv_caches[layer_name] = kv_caches[target_layer_name]

    # 4. 绑定 KV Cache 到模型前后文
    from vllm.v1.worker.utils import bind_kv_cache

    num_attn_module = 2 if self.model_config.hf_text_config.model_type == "longcat_flash" else 1
    bind_kv_cache(
        kv_caches,
        self.compilation_config.static_forward_context,
        self.kv_caches,
        num_attn_module
    )

    return kv_caches

6.2 bind_kv_cache 函数详解

vllm/vllm/v1/worker/utils.py:271

函数签名与职责

def bind_kv_cache(
    kv_caches: dict[str, torch.Tensor],
    forward_context: dict[str, Attention],
    runner_kv_caches: list[torch.Tensor],
    num_attn_module: int = 1,
) -> None:
    """
    Bind the allocated KV cache to both ModelRunner and forward context so
    that the KV cache can be used in the forward pass.

    This function:
      1) Fills the ModelRunner's kv cache list (`runner_kv_caches`) with
         kv_caches.
      2) Associates each attention layer in the `forward_context` with its
         corresponding KV cache in kv_caches.

    Args:
        kv_caches: The allocated kv_caches with layer names as keys.
        forward_context: The global forward context containing all Attention
            layers with layer names as keys.
        runner_kv_caches: The kv_cache declared by ModelRunner.
        num_attn_module: Number of attention modules per layer (1 for standard,
            2 for encoder-decoder models like longcat_flash)
    """

bind_kv_cache 的双重职责

  1. 填充 ModelRunner 的 KV Cache 列表：将按层组织的 KV Cache 张量转换为按索引排序的列表
  2. 绑定到前后文：将每个 Attention 层与对应的 KV Cache 关联，以便在 forward pass 中访问

步骤 1：绑定到 ModelRunner

extract_layer_index 函数

extract_layer_index 从层名中提取层索引。层名通常遵循以下格式：

  • model.layers.0.self_attn → layer_index = 0
  • model.layers.1.cross_attn → layer_index = 1
# 示例：层名到索引的映射
# "model.layers.0.self_attn" → 0
# "model.layers.1.self_attn" → 1
# "model.layers.10.self_attn" → 10

步骤 2：绑定到前后文

# Bind kv_caches to forward context
for layer_name, kv_cache in kv_caches.items():
    # NOTE: Use list because of v0 PP virtual engine.
    forward_context[layer_name].kv_cache = [kv_cache]

这一步将 KV Cache 直接绑定到每个 Attention 层的 kv_cache 属性上。在模型的前向传播中，Attention 层会通过这个属性访问 KV Cache。

为什么使用列表？

kv_cache 被赋值为一个列表 [kv_cache] 而不是单个张量。这是为了支持 v0 版本的 Pipeline Parallelism 虚拟引擎，该引擎可能在同一层使用多个 KV Cache。

6.3 KV Transfer 注册

vllm_ascend/worker/model_runner_v1.py:2459

如果启用了 KV Transfer 功能（用于分布式推理中的 KV Cache 传输），需要将 KV Cache 注册到传输组。

# 在 initialize_kv_cache 中
kv_caches = self.initialize_kv_cache_tensors(kv_cache_config)

# 注册 KV Cache 到 KV Transfer 组（如果启用）
if has_kv_transfer_group():
    get_kv_transfer_group().register_kv_caches(kv_caches)

KV Transfer 的作用

KV Transfer 用于在以下场景中高效传输 KV Cache：

  • Prefill Disaggregation：将 Prefill 阶段生成的 KV Cache 传输到解码设备
    • 分布式推理：在多个设备间共享 KV Cache
  • KV Cache 卸载：将 KV Cache 卸载到 CPU 或其他存储介质
