05 - 特殊注意力机制支持

在 vLLM Ascend 中，除了标准的 Attention 机制外，还支持多种特殊的注意力机制，包括 MLA（Multi-Latent Attention）、稀疏注意力（Sparse Attention）和 Mamba 线性注意力。本章将详细分析这些特殊注意力机制的 KV Cache 处理方式。

5.1 MLA（Multi-Latent Attention）支持

vllm_ascend/worker/model_runner_v1.py:2540

MLA 简介

MLA 是 DeepSeek 模型采用的注意力机制，通过将 KV Cache 分为两个低秩潜在空间来减少内存占用：

  • NoPE Cache：无位置编码的潜在向量，维度为 kv_lora_rank
  • RoPE Cache：带旋转位置编码的向量，维度为 qk_rope_head_dim

MLA 内存分配

if self.model_config.use_mla:
    # 计算总 head size（NoPE + RoPE）
    head_size = (
        self.model_config.hf_text_config.qk_rope_head_dim
        + self.model_config.hf_text_config.kv_lora_rank
    )

# 对于 MLA 模型，K/V 分割比例根据 head dim 计算
k_tensor_split_factor = head_size / self.model_config.hf_text_config.kv_lora_rank
v_tensor_split_factor = head_size / self.model_config.hf_text_config.qk_rope_head_dim

MLA KV Cache 形状重塑

# MLA 模型：K Cache (NoPE) 和 V Cache (RoPE) 形状不同
mla_num_blocks, mla_block_size, num_kv_heads, _ = kv_cache_shape

# k_cache: nope_cache (kv_lora_rank)
k_shape = [
    mla_num_blocks,
    mla_block_size,
    num_kv_heads,
    self.model_config.hf_text_config.kv_lora_rank,
]

# v_cache: rope_cache (qk_rope_head_dim)
v_shape = [
    mla_num_blocks,
    mla_block_size,
    num_kv_heads,
    self.model_config.hf_text_config.qk_rope_head_dim,
]

# 执行视图转换
k_cache = raw_k_tensor.view(dtype).view(k_shape)
v_cache = raw_v_tensor.view(dtype).view(v_shape)

MLA 的优势

相比于标准 Attention，MLA 通过低秩分解显著减少了 KV Cache 的内存占用：

  • 标准 Attention：KV Cache = 2 × num_kv_heads × head_size（通常 head_size=256）
  • MLA：KV Cache = kv_lora_rank + qk_rope_head_dim（通常 512+64=576）

对于 DeepSeek V2/V3 等模型，head_size 可能达到 512 或更高，MLA 可以节省约 50% 的 KV Cache 内存。

5.2 稀疏注意力（Sparse Attention）支持

vllm_ascend/worker/model_runner_v1.py:2552

稀疏注意力简介

稀疏注意力是 DeepSeek V3.2 引入的注意力机制，通过将注意力头分为不同的组，每组使用不同的 KV Cache 策略，进一步减少内存占用。

稀疏 KV Cache 比例

elif self.use_sparse:
    # DeepSeek V3.2: 根据稀疏比例分割 KV Cache
    sparse_sum_head_size = sum(self._get_sparse_kv_cache_ratio())

    # 计算 K/V/DSA_K 的分割因子
    k_tensor_split_factor, v_tensor_split_factor, dsa_k_cache_factor = [
        sparse_sum_head_size / ratio
        for ratio in self._get_sparse_kv_cache_ratio()
    ]

    # 计算 DSA K Cache 大小
    dsa_k_cache_size = int(kv_cache_tensor.size // dsa_k_cache_factor)

DSA K Cache 处理

# 分配 DSA K Cache 张量（如果启用稀疏注意力）
if dsa_k_cache_factor is not None:
    # 无 KV Transfer
    if self.vllm_config.kv_transfer_config is None:
        dsa_k_cache_tensor = torch.zeros(
            dsa_k_cache_size, dtype=torch.int8, device=self.device
        )
    else:
        # 有 KV Transfer：需要 2M 对齐
        dsa_k_cache_tensor = torch.zeros(
            dsa_k_cache_size + alignment, dtype=torch.int8, device=self.device
        )
        dsa_k_cache_tensor = self._align_memory(
            dsa_k_cache_tensor, alignment
        )[:dsa_k_cache_size]

# 重塑 DSA K Cache
if self.use_sparse and raw_dsa_k_tensor is not None:
    index_head_dim = self._get_sparse_kv_cache_ratio()[-1]
    dsa_k_cache_shape = (
        num_blocks,
        kv_cache_spec.block_size,
        kv_cache_spec.num_kv_heads,
        index_head_dim,
    )
    dsa_k_cache = raw_dsa_k_tensor.view(dtype).view(dsa_k_cache_shape)
    kv_caches[layer_name] = (k_cache, v_cache, dsa_k_cache)

稀疏 KV Cache 比例

_get_sparse_kv_cache_ratio() 返回一个三元组，表示 K/V/DSA_K 的头维度比例。例如：

[128, 128, 64]  # K: 128 heads, V: 128 heads, DSA_K: 64 heads

总和为 320，意味着 K/V 各占总内存的 128/320 = 40%，DSA_K 占 20%。

5.3 Mamba 线性注意力支持

vllm_ascend/worker/model_runner_v1.py:2521

Mamba 简介

Mamba 是一种状态空间模型（SSM），使用线性注意力机制替代传统的 Transformer 注意力。它的 KV Cache 结构完全不同，包含卷积状态（conv_state）和 SSM 状态（ssm_state）。

Mamba KV Cache 分配

if "linear_attn" in layer_name and layer_name not in kv_cache_raw_tensors:
    # for mamba linear attention
    if self.vllm_config.kv_transfer_config is None:
        # 无 KV Transfer：直接分配
        tensor = torch.zeros(kv_cache_tensor.size, dtype=torch.int8, device=self.device)
    else:
        # 有 KV Transfer：需要 2M 对齐
        cache_size_aligned = kv_cache_tensor.size + alignment
        tensor = torch.zeros(cache_size_aligned, dtype=torch.int8, device=self.device)
        tensor = self._align_memory(tensor, alignment)[: kv_cache_tensor.size]

    # 在同一个组内的所有 linear_attn 层共享该张量
    for layer_name_inner in kv_cache_tensor.shared_by:
        if "linear_attn" in layer_name_inner:
            kv_cache_raw_tensors[layer_name_inner] = tensor

MambaSpec 重塑流程

elif isinstance(kv_cache_spec, MambaSpec):
    raw_tensor = kv_cache_raw_tensors[layer_name]
    assert raw_tensor is not None
    assert raw_tensor.numel() % kv_cache_spec.page_size_bytes == 0

    num_blocks = raw_tensor.numel() // kv_cache_spec.page_size_bytes
    assert num_blocks >= kv_cache_config.num_blocks

    # Mamba 状态张量列表
    state_tensors = []
    target_idx = 0
    start_idx = 0

    # 遍历所有状态形状和数据类型
    # 正常情况下，这个循环包含 conv state 和 ssm state
    # 某些特殊模型可能只有 conv state
    for shape, dtype in zip(kv_cache_spec.shapes, kv_cache_spec.dtypes):
        target_shape = (num_blocks, *shape)

        # 计算目标索引位置
        target_idx += torch.prod(torch.tensor(target_shape)).item()

        # 从原始张量中切片出当前状态张量
        tensor = raw_tensor.view(dtype)[start_idx:target_idx].view(target_shape)
        start_idx = target_idx

        state_tensors.append(tensor)

    # 返回状态张量列表
    kv_caches[layer_name] = state_tensors

5.4 跨层 KV Cache 共享

vllm_ascend/worker/model_runner_v1.py:2486

某些模型架构中，不同的层可能共享相同的 KV Cache。这种机制可以减少重复内存分配，提高内存效率。

# Set up cross-layer KV cache sharing
for layer_name, target_layer_name in self.shared_kv_cache_layers.items():
    logger.debug("%s reuses KV cache of %s", layer_name, target_layer_name)
    kv_caches[layer_name] = kv_caches[target_layer_name]

共享机制说明

shared_kv_cache_layers 是一个字典，键为当前层名，值为目标层名。通过这种方式，多个层可以共享同一块 KV Cache 内存，适用于以下场景：

  • 循环结构：某些模型使用循环层，多次复用同一层权重
  • 编码器 - 解码器共享：某些架构中编码器和解码器共享 KV Cache
