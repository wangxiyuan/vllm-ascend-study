4.1 _reshape_kv_cache_tensors 方法概览

_reshape_kv_cache_tensors 方法将上一步分配的原始 int8 缓冲区转换为具有正确形状和数据类型的 KV Cache 张量。这个过程中，原始的一维字节数组被重塑为多维张量，以便模型能够正确访问和使用。

4.3 AttentionSpec 处理流程

步骤 1：提取原始张量

if isinstance(kv_cache_spec, AttentionSpec):
    raw_dsa_k_tensor = None

    # 标准注意力：提取 K/V 两个张量

        sum_page_size_bytes = raw_k_tensor.numel() + raw_v_tensor.numel()

    # 验证张量有效性
    assert raw_k_tensor is not None
    assert raw_v_tensor is not None
    assert sum_page_size_bytes % kv_cache_spec.page_size_bytes == 0

page_size_bytes 的含义

page_size_bytes 表示一个 KV Cache 块（page）所需的字节数。计算公式为：

page_size_bytes = block_size × num_kv_heads × head_size × element_size(dtype)

对于 MLA，还需要考虑 kv_lora_rank 和 qk_rope_head_dim 的分别计算。

步骤 2：计算 num_blocks

# 计算可以分配的 KV Cache 块数量
num_blocks = sum_page_size_bytes // kv_cache_spec.page_size_bytes

# 验证 num_blocks 不小于配置中指定的最小值
# `num_blocks` is the number of blocks the model runner can use.
# `kv_cache_config.num_blocks` is the number of blocks that
# KVCacheManager may allocate.
#
# Since different GPUs may have different number of layers and
# different memory capacities, `num_blocks` can be different on

# the min of all `num_blocks`. Verify it here.

步骤 4：重塑 K/V Cache 张量

dtype = kv_cache_spec.dtype

# 根据是否使用 MLA 确定 K/V Cache 形状
if not self.model_config.use_mla:
    # 非 MLA 模型：K/V Cache 形状相同
    k_shape = kv_cache_shape[1:]
    v_shape = k_shape
else:
    # MLA 模型：K Cache (NoPE) 和 V Cache (RoPE) 形状不同
    mla_num_blocks, mla_block_size, num_kv_heads, _ = kv_cache_shape

    # k_cache: nope_cache (kv_lora_rank)
    k_shape = [
        mla_num_blocks,
        mla_block_size,
        num_kv_heads,
        self.model_config.hf_text_config.kv_lora_rank,
    ]

    # v_cache: rope_cache (qk_rope_head_dim)
    v_shape = [
        mla_num_blocks,
        mla_block_size,
        num_kv_heads,
        self.model_config.hf_text_config.qk_rope_head_dim,
    ]

# 执行视图转换
k_cache = raw_k_tensor.view(dtype).view(k_shape)
v_cache = raw_v_tensor.view(dtype).view(v_shape)

MLA 的 NoPE/RoPE 分离

在 DeepSeek MLA 机制中，KV Cache 被分为两个独立的部分：

  • K Cache (NoPE)：存储无位置编码的潜在向量，维度为 kv_lora_rank（通常为 512）
  • V Cache (RoPE)：存储带旋转位置编码的查询/键向量，维度为 qk_rope_head_dim（通常为 64-128）

这种设计使得 MLA 可以显著减少 KV Cache 的内存占用，同时保持模型质量。

步骤 5：处理稀疏注意力（DSA）

if self.use_sparse and raw_dsa_k_tensor is not None:

    dsa_k_cache_shape = (

        kv_cache_spec.block_size,
        kv_cache_spec.num_kv_heads,

else:

4.4 MambaSpec 处理流程

对于 Mamba 等状态空间模型的线性注意力机制，KV Cache 的处理方式不同：

elif isinstance(kv_cache_spec, MambaSpec):
    raw_tensor = kv_cache_raw_tensors[layer_name]
    assert raw_tensor is not None
    assert raw_tensor.numel() % kv_cache_spec.page_size_bytes == 0

    num_blocks = raw_tensor.numel() // kv_cache_spec.page_size_bytes
    assert num_blocks >= kv_cache_config.num_blocks

    # Mamba 状态张量列表
    state_tensors = []
    target_idx = 0
    start_idx = 0

    # 遍历所有状态形状和数据类型
    for shape, dtype in zip(kv_cache_spec.shapes, kv_cache_spec.dtypes):
        # 正常情况下，这个循环包含 conv state 和 ssm state
        # 某些特殊模型可能只有 conv state
        target_shape = (num_blocks, *shape)

        # 计算目标索引位置
        target_idx += torch.prod(torch.tensor(target_shape)).item()

        # 从原始张量中切片出当前状态张量
        tensor = raw_tensor.view(dtype)[start_idx:target_idx].view(target_shape)
        start_idx = target_idx

        state_tensors.append(tensor)

    # 返回状态张量列表
    kv_caches[layer_name] = state_tensors
