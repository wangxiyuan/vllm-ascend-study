3. 注意力状态判断逻辑

def _build_attn_state(self, num_reqs, num_scheduled_tokens, num_valid_tokens):
    # 首次 prefill，无 KV 缓存
    if np.all(self.input_batch.num_computed_tokens_cpu[:num_reqs] == 0):
        return AscendAttentionState.PrefillNoCache

    # 纯 decode 阶段：每个请求只调度 1 个令牌
    if np.all(num_scheduled_tokens == 1):
        if self.speculative_config and self.speculative_config.method == "mtp":
            return AscendAttentionState.SpecDecoding
        return AscendAttentionState.DecodeOnly

    # 投机解码：每个请求只有 1 个有效令牌
    if np.all(num_valid_tokens == 1):
        if self.speculative_config:
            return AscendAttentionState.SpecDecoding
        return AscendAttentionState.ChunkedPrefill

    # 分块 prefill
    if self.scheduler_config.enable_chunked_prefill:
        return AscendAttentionState.ChunkedPrefill

    # Prefill 有缓存命中
    return AscendAttentionState.PrefillCacheHit

注意力状态选择策略

状态 条件 优化内核
PrefillNoCache 所有请求 num_computed_tokens == 0 FlashAttention Prefill
DecodeOnly 所有请求 num_scheduled_tokens == 1 PagedAttention Decode
ChunkedPrefill 启用分块 prefill 且混合批次 FlashAttention + PagedAttention
SpecDecoding 投机解码模式 专门的投机解码内核

PCP (Prefill Context Parallel)

PCP 是 Ascend 特有的优化，用于在 prefill 阶段实现上下文并行。 它将长序列分割到多个 NPU 上并行处理，然后通过 all-gather 合并结果。
