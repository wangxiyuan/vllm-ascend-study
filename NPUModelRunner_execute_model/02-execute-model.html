@torch.inference_mode()
def execute_model(
    self,
    scheduler_output: "SchedulerOutput",
    intermediate_tensors: IntermediateTensors | None = None,
) -> ModelRunnerOutput | IntermediateTensors | None:

参数说明

参数 类型 说明
scheduler_output SchedulerOutput 调度器输出，包含调度的请求、令牌数、KV 缓存块等信息
intermediate_tensors IntermediateTensors | None 流水线并行时的中间张量，非首阶段使用

返回值

返回类型 场景
None 正常执行完成，状态保存在 execute_model_state 中，等待 sample_tokens 调用
IntermediateTensors 流水线并行的非最后阶段，返回中间张量给下一阶段
ModelRunnerOutput 无令牌调度时的空输出

两阶段执行模式

execute_model 和 sample_tokens 采用两阶段执行模式：

  1. execute_model 完成模型前向传播，返回 None
  2. sample_tokens 完成采样，返回最终输出

这种设计允许异步调度在模型执行期间并行处理采样。

set_ascend_forward_context

这是 Ascend 特有的上下文管理器，用于设置前向传播所需的 Ascend 特定参数， 包括 ACL Graph 模式、批次描述符、MoE 通信方法等。

阶段 7: 后处理与状态保存

# 处理 PCP (Prefill Context Parallel)
if self.pcp_size > 1:
    hidden_states = self.pcp_manager.get_restore_hidden_states(hidden_states)

# 非最后 PP 阶段
if not self.broadcast_pp_output:
    if not get_pp_group().is_last_rank:
        assert isinstance(hidden_states, IntermediateTensors)
        hidden_states.kv_connector_output = kv_connector_output
        return hidden_states

    # Pooling 模型
    if self.is_pooling_model:
        output = self._pool(hidden_states, ...)

CUDA Graph 模式

_determine_batch_execution_and_padding 方法会根据批次特征确定 CUDA Graph 模式：

模式 说明
NONE 不使用 CUDA Graph，eager 模式执行
PIECEWISE 部分 CUDA Graph，某些内核使用图捕获
FULL 完整 CUDA Graph，整个前向传播使用图捕获
