NPUModelRunner.execute_model 代码分析

vllm_ascend/worker/model_runner_v1.py

概述

本文档详细分析 vLLM-Ascend 项目中 NPUModelRunner 类的 execute_model 方法。 该方法是 NPU 上模型推理的核心执行入口，负责将调度器输出的请求转换为模型可处理的输入张量， 执行模型前向传播，并为后续的 token 采样做准备。

类继承关系

┌─────────────────────┐ │ GPUModelRunner │ ← vLLM 基类 (vllm/v1/worker/gpu_model_runner.py) └──────────┬──────────┘ │ 继承 ▼ ┌─────────────────────┐ │ NPUModelRunner │ ← Ascend 适配实现 (vllm_ascend/worker/model_runner_v1.py) └─────────────────────┘

执行流程概览

flowchart TD A[execute_model 入口] --> B[_update_states 更新状态] B --> C[_prepare_inputs 准备输入] C --> D[_build_attention_metadata 构建注意力元数据] D --> E[_preprocess 预处理] E --> F[_model_forward 模型前向传播] F --> G{是否为最后 PP 阶段?} G -->|否| H[返回 IntermediateTensors] G -->|是| I[保存状态到 execute_model_state] I --> J[返回 None] J --> K[sample_tokens 采样] K --> L[返回 ModelRunnerOutput]

关键概念

  • NPU: Neural Processing Unit，华为昇腾 AI 处理器
  • ACL Graph: 昇腾计算图，类似于 CUDA Graph，用于优化推理性能
  • SplitFuse: 将 prefill 和 decode 请求合并到同一批次处理
  • PCP: Prefill Context Parallel，prefill 阶段的上下文并行
  • 投机解码: 使用小型 drafter 模型预测 token，加速推理
