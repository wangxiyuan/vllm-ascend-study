第五章：_build_attention_metadata - 注意力元数据构建

vllm_ascend/worker/model_runner_v1.py:1847-2060

方法概述

_build_attention_metadata 方法是模型推理过程中至关重要的步骤， 负责构建注意力机制所需的元数据。这些元数据包括 KV 缓存的位置信息、 序列长度、block 表等，直接影响注意力计算的效率和正确性。

核心数据结构

AscendCommonAttentionMetadata

这是 Ascend 平台上注意力元数据的核心数据结构：

@dataclass
class AscendCommonAttentionMetadata(CommonAttentionMetadata):
    """
    批次级别的注意力元数据，跨层和后端共享。
    AttentionMetadataBuilder 实例使用它构建每层元数据。
    """
    # CPU 上的序列长度张量
    seq_lens_cpu: torch.Tensor = None

    # CPU 上已计算 token 数量
    num_computed_tokens_cpu: torch.Tensor = None

    # 每个请求的解码 token 数（投机解码场景）
    decode_token_per_req: int = 1

    # 实际查询序列长度列表
    actual_seq_lengths_q: list[int] = field(default_factory=list)

    # NPU 上的位置索引张量
    positions: torch.Tensor = None

    # 当前注意力状态
    attn_state: Any = None

    # Graph 模式的填充大小
    graph_pad_size: int = -1

    # PCP 元数据
    prefill_context_parallel_metadata: AscendPrefillContextParallelMetadata | None = None

2. Block Table 和 Slot Mapping 获取

Block Table

Block Table 是 KV Cache 管理的核心数据结构，记录了每个请求的 KV Cache 在物理内存中的位置。每个请求对应一个 block 列表，每个 block 存储固定数量 的 token 的 KV 数据。 

def _get_block_table_and_slot_mapping(kv_cache_gid: int):
    kv_cache_spec = kv_cache_groups[kv_cache_gid].kv_cache_spec

    if isinstance(kv_cache_spec, EncoderOnlyAttentionSpec):
        # Encoder-only 模型使用零张量
        blk_table_tensor = torch.zeros((num_reqs_padded, 1), ...)
        slot_mapping = torch.zeros((num_tokens_padded,), ...)
    else:
        # 从 input_batch 获取 block table
        blk_table = self.input_batch.block_table[kv_cache_gid]
        slot_mapping = blk_table.slot_mapping.gpu[:maybe_pcp_full_tokens]
        blk_table_tensor = blk_table.get_device_tensor()[:maybe_num_reqs_padded]

        # 填充未使用位置为 -1
        slot_mapping[num_tokens:num_tokens_padded].fill_(-1)
        blk_table_tensor[num_reqs:num_reqs_padded].fill_(0)

    return blk_table_tensor, slot_mapping

3. Prefill Context Parallel (PCP) 元数据

PCP (Prefill Context Parallel)

PCP 是 prefill 阶段的上下文并行技术，用于处理超长序列。 通过将长序列分割到多个设备并行处理，提升长序列处理能力。 

def _get_pcp_metadata(block_table_tensor):
    if not self.use_cp:
        return None, block_table_tensor
    return self.pcp_manager.generate_pcp_metadata(
        num_tokens,
        self.query_lens,
        self.input_batch,
        num_scheduled_tokens_np,
        block_table_tensor,
        num_reqs_padded,
        num_reqs,
    )

┌─────────────────────────────────────────────────────┐ │ KV Cache Group 0 │ ├─────────────────────────────────────────────────────┤ │ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ │ │ Attention │ │ Attention │ │ Attention │ │ │ │ Group 0 │ │ Group 1 │ │ Group N │ │ │ └──────┬──────┘ └──────┬──────┘ └──────┬──────┘ │ │ │ │ │ │ │ ▼ ▼ ▼ │ │ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ │ │ Builder │ │ Builder │ │ Builder │ │ │ │ .build() │ │ .build() │ │ .build() │ │ │ └─────────────┘ └─────────────┘ └─────────────┘ │ └─────────────────────────────────────────────────────┘
def _build_attn_group_metadata(
    kv_cache_gid: int,
    attn_gid: int,
    common_attn_metadata: CommonAttentionMetadata,
    ubid: int | None = None,
) -> None:
    attn_group = self.attn_groups[kv_cache_gid][attn_gid]
    builder = attn_group.get_metadata_builder(ubid or 0)

    if for_cudagraph_capture:
        attn_metadata_i = builder.build_for_cudagraph_capture(common_attn_metadata)
    else:
        attn_metadata_i = builder.build(
            common_prefix_len=cascade_attn_prefix_len,
            common_attn_metadata=common_attn_metadata,
            **extra_attn_metadata_args,
        )

    # 同一组的所有层共享相同元数据
    for layer_name in attn_group.layer_names:
        attn_metadata_dict[layer_name] = attn_metadata_i

  • Block Table：KV Cache 的物理位置映射表
  • Slot Mapping：Token 到 Cache Slot 的映射

  • 投机解码支持：GDN 构建器额外参数
