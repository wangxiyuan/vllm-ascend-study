第四章：Attention 规范生成

Attention.get_kv_cache_spec 方法

vllm/vllm/model_executor/layers/attention/attention.py:519
def get_kv_cache_spec(self, vllm_config: VllmConfig) -> KVCacheSpec:
    # Block size may get updated after model loading, refresh it
    block_size = vllm_config.cache_config.block_size
    # Should not be called for enc-dec or encoder-only attention.
    assert self.attn_type == AttentionType.DECODER
    if self.sliding_window is not None:
        assert not vllm_config.model_config.use_mla, (
            "MLA is not supported for slidingwindow"
        )
        return SlidingWindowSpec(
            block_size=block_size,
            num_kv_heads=self.num_kv_heads,
            head_size=self.head_size,
            dtype=self.kv_cache_torch_dtype,
            sliding_window=self.sliding_window,
        )
    else:
        return FullAttentionSpec(
            block_size=block_size,
            num_kv_heads=self.num_kv_heads,
            head_size=self.head_size,
            head_size_v=self.head_size_v,
            dtype=self.kv_cache_torch_dtype,
        )

方法逻辑分析

1. 获取块大小

block_size = vllm_config.cache_config.block_size

从配置中获取 KV Cache 的块大小。块大小决定了每个 KV Cache 块可以存储多少个 token 的 KV 数据。 常见值为 16 或 64。

2. 类型断言

assert self.attn_type == AttentionType.DECODER

断言当前层是解码器注意力类型。编码器-解码器模型中的编码器注意力或仅编码器注意力 不应该调用此方法，因为它们不需要 KV Cache。

3. 规范类型选择

+-------------------------+ | Attention.get_kv_cache_spec | +-------------------------+ | v +-------------------------+ | sliding_window != None? | +-------------------------+ | | Yes No | | v v +----------------+ +-------------------+ |SlidingWindowSpec| |FullAttentionSpec | +----------------+ +-------------------+ | | v v +----------------+ +-------------------+ | 包含窗口大小 | | 标准 KV Cache | | 参数 | | 规范 | +----------------+ +-------------------+

FullAttentionSpec 详解

标准全注意力规范，适用于大多数 Transformer 模型：

@dataclass(frozen=True, kw_only=True)
class FullAttentionSpec(AttentionSpec):
    """
    When hybrid allocator is disabled and the model contains both full
    attention layers and sliding window attention layers, sliding
    window attention are regarded as full attention in KV cache manager
    (blocks are allocated for all tokens), while computed as sliding window
    attention in model runner.
    In this case, we use FullAttentionSpec and record the sliding window size.
    """

    head_size_v: int = None  # type: ignore[assignment]
    sliding_window: int | None = None
    attention_chunk_size: int | None = None

关键属性

属性 类型 描述
block_size int 每个块的 token 数量
num_kv_heads int KV 头数量（GQA 中可能小于 query 头数）
head_size int Key 的头维度
head_size_v int Value 的头维度（默认与 head_size 相同）
dtype torch.dtype KV Cache 的数据类型

内存计算

def max_memory_usage_bytes(self, vllm_config: VllmConfig) -> int:
    max_model_len = vllm_config.model_config.max_model_len
    dcp_world_size = vllm_config.parallel_config.decode_context_parallel_size
    pcp_world_size = vllm_config.parallel_config.prefill_context_parallel_size
    # Note(hc): each dcp rank only need save
    # (max_model_len//dcp_world_size) tokens locally.
    if dcp_world_size * pcp_world_size > 1:
        max_model_len = cdiv(max_model_len, dcp_world_size * pcp_world_size)
    return cdiv(max_model_len, self.block_size) * self.page_size_bytes

内存使用量计算考虑了解码上下文并行（DCP）和预填充上下文并行（PCP）的影响。 在并行场景下，每个 rank 只需要保存部分 token 的 KV Cache。

SlidingWindowSpec 详解

滑动窗口注意力规范，适用于 Mistral 等使用滑动窗口注意力的模型：

@dataclass(frozen=True, kw_only=True)
class SlidingWindowSpec(AttentionSpec):
    sliding_window: int

    def max_memory_usage_bytes(self, vllm_config: VllmConfig) -> int:
        assert vllm_config.parallel_config.decode_context_parallel_size == 1, (
            "DCP not support sliding window."
        )
        max_model_len = vllm_config.model_config.max_model_len
        max_num_batched_tokens = vllm_config.scheduler_config.max_num_batched_tokens

        # During chunked prefill, we allocate KV cache for the last
        # `self.sliding_window-1` computed tokens plus the newly scheduled
        # tokens. And we won't allocate KV cache for more than `max_model_len`
        # tokens.
        num_tokens = min(
            self.sliding_window - 1 + max_num_batched_tokens, max_model_len
        )

        # +1 here because the sliding window may not start from the beginning
        # of the block. For example, if the block size is 4 and num_token
        # is 4, we need two blocks [XXCD] [EF] to store the sliding
        # window [CDEF] of 6 tokens.
        return (cdiv(num_tokens, self.block_size) + 1) * self.page_size_bytes

滑动窗口内存优化

滑动窗口注意力不需要缓存所有历史 token，只需要缓存最近的 sliding_window 个 token。 这大大减少了 KV Cache 的内存占用，特别是对于长上下文场景。

page_size_bytes 计算

@property
def page_size_bytes(self) -> int:
    return (
        self.block_size
        * self.num_kv_heads
        * (self.head_size + self.head_size_v)
        * get_dtype_size(self.dtype)
    )

一个页面（块）的字节大小计算：

  • block_size: 每个 block 的 token 数
  • num_kv_heads: KV 头数
  • head_size + head_size_v: Key 和 Value 的维度之和
  • get_dtype_size(dtype): 每个元素的字节数
