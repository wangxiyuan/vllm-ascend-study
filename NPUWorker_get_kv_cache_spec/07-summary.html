设计模式总结

1. 委托模式 (Delegation Pattern)

NPUWorker 将具体的规范计算工作委托给 NPUModelRunner， 保持 Worker 的轻量级。

2. 模板方法模式 (Template Method Pattern)

AttentionLayerBase 定义了抽象方法 get_kv_cache_spec， 各具体层类实现各自的版本。

3. 策略模式 (Strategy Pattern)

不同类型的层（Attention、MLA、Mamba）使用不同的策略生成 KV Cache 规范。

内存优化策略

+--------------------------------------------------+ | KV Cache 内存优化策略 | +--------------------------------------------------+ | | | 1. GQA (Grouped Query Attention) | | - num_kv_heads < num_heads | | - 减少 KV Cache 大小 | | | | 2. MLA (Multi-Latent Attention) | | - KV 压缩到低秩空间 | | - 大幅减少内存占用 | | | | 3. Sliding Window Attention | | - 只缓存窗口内的 token | | - 适用于长上下文 | | | | 4. Mamba/SSM | | - 固定大小的状态 | | - 与序列长度无关 | | | +--------------------------------------------------+

Ascend 特定优化

Ascend 平台优化点

  • Sparse MLA：针对 DeepSeek 模型的 Sparse 模式特殊处理
  • ACL Graph：支持 ACL Graph 加速
  • 内存对齐：通过 page_size_padded 支持内存对齐优化

总结

get_kv_cache_spec 方法是 vLLM 内存管理的核心环节， 它通过遍历模型中的注意力类层，收集各层的 KV Cache 规范， 为后续的内存分配和 KV Cache 初始化提供依据。 理解这个流程对于优化 vLLM 的内存使用和支持新的模型架构至关重要。
