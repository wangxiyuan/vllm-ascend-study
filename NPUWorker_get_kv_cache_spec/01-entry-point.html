第一章：入口方法 - NPUWorker::get_kv_cache_spec

方法定义

vllm-ascend/vllm_ascend/worker/worker.py:486
def get_kv_cache_spec(self) -> dict[str, KVCacheSpec]:
    return self.model_runner.get_kv_cache_spec()

方法分析

NPUWorker.get_kv_cache_spec 是一个非常简洁的委托方法， 它直接将调用转发给 self.model_runner。 这种设计遵循了单一职责原则：

  • NPUWorker：负责 Worker 级别的协调和生命周期管理
  • NPUModelRunner：负责具体的模型执行和状态管理

NPUWorker 类结构

调用时机

get_kv_cache_spec 方法在引擎初始化阶段被调用， 用于确定 KV Cache 的内存布局。典型的调用流程如下：

sequenceDiagram participant Engine as EngineCore participant Worker as NPUWorker participant Runner as NPUModelRunner participant Layers as Model Layers Engine->>Worker: determine_available_memory() Worker->>Worker: profile_run() Worker-->>Engine: available_memory_bytes Engine->>Worker: get_kv_cache_spec() Worker->>Runner: get_kv_cache_spec() Runner->>Layers: 遍历各层 Layers-->>Runner: KVCacheSpec Runner-->>Worker: dict[str, KVCacheSpec] Worker-->>Engine: dict[str, KVCacheSpec] Note over Engine: 根据 KVCacheSpec 计算内存分配

返回值说明

方法返回一个字典，键为层名称，值为对应的 KVCacheSpec 对象：

# 示例返回值结构
{
    "model.layers.0.self_attn": FullAttentionSpec(
        block_size=16,
        num_kv_heads=32,
        head_size=128,
        dtype=torch.float16,
        ...
    ),
    "model.layers.1.self_attn": FullAttentionSpec(...),
    # ... 更多层
}

设计模式

这里使用了委托模式（Delegation Pattern）。 Worker 将具体的规范计算工作委托给 ModelRunner， 使得 Worker 保持轻量级，专注于协调工作。

继承关系

NPUWorker 继承自 vLLM 的 WorkerBase 类， get_kv_cache_spec 是 WorkerBase 中定义的抽象方法。
