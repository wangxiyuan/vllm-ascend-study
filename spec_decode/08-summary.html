数据结构与传递路径

阶段 Token 类型 数据结构 传递路径

核心机制

1. 延迟验证

draft model 生成的 token 不会立即使用，而是缓存到 request.spec_token_ids， 下一轮才被 target model 验证。

2. Hidden States 复用

draft model 使用 target model 的 hidden states 作为输入，避免重复计算。

3. Async Scheduling

支持异步调度，draft token 的生成可以和 target model 的计算并行。

4. SpecDecodeMetadata 结构

SpecDecodeMetadata(
    draft_token_ids=draft_token_ids,         # draft token 的 ID
    num_draft_tokens=num_draft_tokens,       # 每个 request 的 draft 数量
    cu_num_draft_tokens=cu_num_draft_tokens, # 累计 draft tokens
    cu_num_sampled_tokens=cu_num_sampled_tokens, # 累计 sampled tokens
    target_logits_indices=target_logits_indices, # target 位置索引
    logits_indices=logits_indices,           # logits 计算位置
    bonus_logits_indices=bonus_logits_indices,   # bonus token 索引
)

关键要点

关键点

  • Speculative Decoding 的核心：用草稿模型快速生成多个 token，用目标模型并行验证
  • 接受率是关键指标：接受率越高，加速效果越好
  • num_speculative_tokens 需要调优：太大导致接受率下降，太小加速效果有限
  • 不同方法适用于不同场景：
    • EAGLE：高接受率，需要额外训练
    • MTP：集成度高，仅支持特定模型
    • Ngram：简单快速，接受率较低
    • Suffix：利用缓存，适合重复模式

vllm-ascend 实现特点

  • EAGLE 方法优化：支持 GPU padded batch 模式，可以使用 GPU sampled tokens 直接作为输入
  • ACL Graph 支持：支持 ACL Graph 捕获和重放，提高推理性能
  • PCP/DCP 分布式支持：支持 Prefill Context Parallel 和 Decode Context Parallel
