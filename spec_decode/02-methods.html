支持的 Speculative 方法

方法对比

方法 是否需要额外模型 优点 缺点
EAGLE 是 (EAGLE head) 高接受率，速度快 需要训练额外 head
EAGLE3 是 (EAGLE3 head) 改进的 EAGLE，更高接受率 需要特定模型支持
MTP 否 (内置) 无需额外模型，集成度高 仅支持特定架构
Ngram 否 无需模型，简单快速 接受率较低
Suffix Decoding 否 利用历史缓存，无训练 需要额外依赖
Draft Model 是 (完整模型) 灵活，可自定义 资源开销大
Medusa 是 (Medusa head) 多 head 并行 需要训练
MLPSpeculator 是 (MLP head) 轻量级 需要训练

EAGLE / EAGLE3

位置: vllm/v1/spec_decode/eagle.py, vllm_ascend/spec_decode/eagle_proposer.py

原理

  • EAGLE 使用一个额外的 head 预测下一个 token 的特征
  • 利用目标模型的隐藏状态进行快速预测
  • EAGLE3 是改进版本，支持更多模型架构

支持的目标模型

MTP (Multi-Token Prediction)

位置: vllm/config/speculative.py

原理

  • MTP 是模型内置的多 token 预测层
  • 某些模型架构（如 DeepSeek、Qwen3.5、Pangu 等）原生支持
  • 无需额外的 draft 模型

配置转换

def hf_config_override(hf_config):
    """自动检测并配置 MTP"""
    if hf_config.model_type in ("deepseek_v3", "deepseek_v32"):
        hf_config.model_type = "deepseek_mtp"
    if hf_config.model_type == "qwen3_5_moe":
        hf_config.model_type = "qwen3_5_mtp"
    # ...更多 MTP 类型
    return hf_config

Draft Model

原理

  • 使用一个独立的较小模型作为 draft 模型
  • 最灵活但资源开销最大

配置示例

speculative_config = {
    "method": "draft_model",
    "model": "TinyLlama/TinyLlama-1.1B-Chat-v1.0",
    "num_speculative_tokens": 4,
}
