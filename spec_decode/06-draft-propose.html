Draft Token 生成

概述

Draft Proposer 负责生成下一轮的草稿 token。在 RejectionSampler 输出采样的 token 后， Proposer 使用 target model 的 hidden states 和 sampled tokens 作为输入， 快速生成多个候选 token 供下一轮验证。

        scheduler_output,
        spec_decode_metadata,
        spec_decode_common_attn_metadata,

        scheduler_output.total_num_scheduled_tokens,
        hidden_states,                  # target model 的 hidden states
        attn_metadata,
        aux_hidden_states,
        sample_hidden_states,
    )
    self._copy_draft_token_ids_to_cpu(scheduler_output)

策略选择

根据 speculative decoding 类型选择不同的策略： 

# 根据 speculative decoding 类型选择不同策略
with record_function_or_nullcontext("draft_token"):
    if self.speculative_config:
        use_padded_batch_for_eagle = (
            self.speculative_config
            and self.speculative_config.use_eagle()
            and not self.speculative_config.disable_padded_drafter_batch
        )
        if use_padded_batch_for_eagle:
            # EAGLE 可以使用 GPU sampled tokens，无需等待 bookkeeping 完成
            propose_draft_token_ids(sampler_output.sampled_token_ids)
        if self.speculative_config and not use_padded_batch_for_eagle:
            # ngram 和其他方法使用 CPU sampled tokens
            propose_draft_token_ids(valid_sampled_token_ids)

EAGLE Draft Model 前向传播

def _propose(
    self,
    target_token_ids: torch.Tensor,      # target model 的输入 tokens
    target_positions: torch.Tensor,      # 位置信息
    target_hidden_states: torch.Tensor,  # ← 来自 target model 的 hidden states
    next_token_ids: torch.Tensor,        # ← 上一轮采样的 tokens
    last_token_indices: torch.Tensor,    # 每个 request 的最后一个 token 索引
    common_attn_metadata: AscendCommonAttentionMetadata,
    sampling_metadata: SamplingMetadata,
    req_scheduled_tokens: dict[str, int],
    num_scheduled_tokens: int,
) -> list[list[int]]:
    """
    EAGLE 方法生成下一轮的 draft tokens
    """
    num_tokens = target_token_ids.shape[0]

    # 1. 设置输入（使用 buffer 避免重复分配）
    self.input_ids[:num_tokens] = target_token_ids
    self.hidden_states[:num_tokens] = target_hidden_states

    # 2. 构建注意力元数据
    # 对于多步 speculative decoding，需要为每一步构建 attn_metadata
    multi_steps_attn_metadata = [...]

    # 3. Draft Model 多步前向传播
    # _run_merged_draft 一次性生成所有 draft tokens
    draft_token_ids = self._runnable(
        num_input_tokens=num_input_tokens,
        batch_size=batch_size,
        last_token_indices=last_token_indices,
        target_positions=target_positions,
        inputs_embeds=inputs_embeds,
        multi_steps_attn_metadata=multi_steps_attn_metadata,
        ...
    )
    # draft_token_ids shape: (batch_size, num_spec_tokens)

    # 4. 返回 draft tokens 列表
    # 例如：[[d1', d2', d3'], [d1'', d2'', d3''], ...]
    return draft_token_ids.cpu().tolist()

Hidden States 复用

EAGLE 方法的关键优化是复用 target model 的 hidden states，避免重复计算： 

# 1. 从 target model 获取 hidden states
target_hidden = target_model(input_ids)

# 2. Draft model 使用 target hidden states 作为输入
for i in range(num_spec_tokens):
    draft_logits = draft_head(target_hidden)
    draft_token = draft_logits.argmax()
    draft_tokens.append(draft_token)

ACL Graph 支持

vllm-ascend 为 EAGLE 方法提供了 ACL Graph 支持，通过捕获和重放提高推理性能：

  • 为 draft model 单独管理 attention metadata
  • 支持多步 speculative decoding 的图捕获
  • 优化了 GPU 内存使用
