核心组件

1. Scheduler（调度器）

位置: vllm/v1/core/sched/scheduler.py

Scheduler 负责管理请求的调度，包括 draft tokens 的管理。 它维护每个请求的 spec_token_ids 状态，用于存储待验证的草稿 token。 

class Request:
    self.spec_token_ids: list[int] = []  # 存储 draft model 生成的待验证 tokens
    self.num_computed_tokens = 0  # 已计算的 token 数
    self.num_tokens = len(prompt) + len(output)  # prompt + output tokens

2. Draft Proposer（草稿生成器）

位置: vllm/v1/spec_decode/

Draft Proposer 负责生成草稿 token，支持多种实现：

  • EAGLEProposer: 使用 EAGLE head 进行预测
  • MTPProposer: 使用模型内置的 MTP 层
  • NgramProposer: 基于 n-gram 匹配
  • SuffixDecodingProposer: 基于后缀树缓存

3. Target Model（目标模型）

位置: vllm_ascend/worker/model_runner_v1.py

Target Model 是原始的大语言模型，负责对草稿 token 进行验证。 它一次性计算所有位置的 logits，包括草稿 token 和新 token 的位置。

4. RejectionSampler（拒绝采样器）

位置: vllm/v1/sample/rejection_sampler.py

RejectionSampler 实现拒绝采样算法，根据 target model 的 logits 决定接受或拒绝草稿 token。

关键数据结构

SpecDecodeMetadata

位置: vllm/v1/spec_decode/metadata.py

@dataclass
class SpecDecodeMetadata:
    draft_token_ids: torch.Tensor        # [num_tokens] 草稿 token
    num_draft_tokens: list[int]          # [batch_size] 每个请求的草稿数
    cu_num_draft_tokens: torch.Tensor    # [batch_size] 累积草稿数
    cu_num_sampled_tokens: torch.Tensor  # [batch_size] 累积采样数
    target_logits_indices: torch.Tensor  # [num_tokens] target logits 索引
    bonus_logits_indices: torch.Tensor   # [batch_size] bonus logits 索引
    logits_indices: torch.Tensor         # [num_tokens + batch_size] 全部索引

SchedulerOutput

位置: vllm/v1/core/sched/output.py

@dataclass
class SchedulerOutput:
    scheduled_spec_decode_tokens: dict[str, list[int]]  # req_id -> [draft_token_1, ...]

工作流程概览

第 1 轮（无历史 draft tokens）:
  Scheduler: scheduled_spec_decode_tokens = {} (空)
  execute_model: target model 只计算 1 个 token（普通 decode）
  sample_tokens:
    - rejection_sampler: 采样 1 个 bonus token
    - drafter: 生成 3 个 draft tokens [d1, d2, d3]
    - 保存到 scheduler: request.spec_token_ids = [d1, d2, d3]

第 2 轮（验证第 1 轮生成的 draft tokens）:
  Scheduler: scheduled_spec_decode_tokens = {req_id: [d1, d2, d3]}
  execute_model:
    - target model 计算 4 个位置（验证 d1, d2, d3 + 新 1 个 bonus 位置）
    - SpecDecodeMetadata 包含 3 个 draft tokens 的索引信息
  sample_tokens:
    - rejection_sampler: 假设 accept [d1, d2]，reject d3，resample 1 个
    - 输出：[d1, d2, r1] (2 个 accepted + 1 个 resampled)
    - drafter: 基于 [d1, d2, r1] 生成下一轮 draft [d1', d2', d3']

第 3 轮（验证第 2 轮生成的 draft tokens）:
  Scheduler: scheduled_spec_decode_tokens = {req_id: [d1', d2', d3']}
  ... 循环继续
