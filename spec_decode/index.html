vLLM Speculative Decoding - 推测解码详解

概述

Speculative Decoding（推测解码，简称 Spec Decode）是一种加速 LLM 推理的技术。 其核心思想是使用一个较小的"草稿模型"（draft model）或更轻量的方法快速生成多个候选 token， 然后使用原始"目标模型"（target model）并行验证这些候选 token。

核心优势

传统解码：    [T1] → [T2] → [T3] → [T4] → [T5]  (5 次串行 forward)
Spec Decode:  [D1,D2,D3,D4] → 验证 → [T1,T2✓,T3✓,T4✗,T5]  (2 次 forward, 接受 3 个)

关键指标：

  • Acceptance Rate（接受率）：草稿 token 被接受的比例
  • Acceptance Length（接受长度）：每次迭代平均接受的 token 数（含 bonus token）
  • Speedup：理想情况下可达 num_spec_tokens + 1 倍加速

核心概念

术语 含义
Draft Tokens（草稿 token） 由草稿模型或推测方法生成的候选 token
Target Tokens（目标 token） 由目标模型生成的 token
Accepted Tokens（接受 token） 通过验证被接受的草稿 token
Rejected Tokens（拒绝 token） 未通过验证被拒绝的草稿 token
Bonus Token（奖励 token） 当所有草稿 token 都被接受时，从目标分布额外采样的一个 token
Recovered Tokens（恢复 token） 当草稿 token 被拒绝时，根据调整后分布采样的 token

Rejection Sampling（拒绝采样）算法

vLLM 实现了标准的拒绝采样算法，核心公式：

对于每个草稿 token γ，接受概率 = min(1, P_target(γ) / P_draft(γ))

算法流程：

  1. 草稿模型生成 k 个 token: γ₁, γ₂, ..., γₖ
  2. 目标模型计算所有位置的分布：P_target(·|x, γ₁:i-1)
  3. 对每个位置进行拒绝采样：
    • 以概率 min(1, P_target(γᵢ) / P_draft(γᵢ)) 接受
    • 如果拒绝，从调整分布采样恢复 token
  4. 如果全部接受，额外采样一个 bonus token

支持的方法对比

方法 是否需要额外模型 优点 缺点
EAGLE 是 (EAGLE head) 高接受率，速度快 需要训练额外 head
EAGLE3 是 (EAGLE3 head) 改进的 EAGLE，更高接受率 需要特定模型支持
MTP 否 (内置) 无需额外模型，集成度高 仅支持特定架构
Ngram 否 无需模型，简单快速 接受率较低
Suffix Decoding 否 利用历史缓存，无训练 需要额外依赖
Draft Model 是 (完整模型) 灵活，可自定义 资源开销大
